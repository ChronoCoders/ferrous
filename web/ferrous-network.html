Core Properties
01

Quantum Resistant

ECDSA is vulnerable to Shor's algorithm on a sufficiently powerful quantum computer. Ferrous replaces it with CRYSTALS-Dilithium, a NIST-standardized lattice-based signature scheme with no known quantum attack.

CRYSTALS-Dilithium
02

Private by Default

Transaction amounts and sender identities are confidential on-chain. Ring Confidential Transactions hide amounts using Pedersen commitments and obscure the sender within a ring of decoys.

Ring CT
03

Proof-of-Work

No validators, no staking, no governance tokens. SHA-256d PoW with per-block difficulty adjustment. A decade of adversarial stress-testing across the largest decentralized network in existence.

SHA-256d
Why Now

NIST finalized post-quantum cryptography standards in 2024. The migration window for existing systems is now open — not theoretical.

Retrofitting quantum resistance onto a live blockchain with millions of existing keys is a coordination problem with no clean solution. A new chain has no legacy debt.

Ferrous is built for the threat that is coming, not the one that has already passed.

Run a Node

A Ferrous full node validates the chain independently, relays transactions, and strengthens network decentralization. No registration required.

01

Requirements

Linux x86_64. 2GB RAM minimum, 4GB recommended. 50GB disk for initial sync. A stable internet connection with a publicly reachable IP on port 8333.
02

Download

Grab the latest binary from GitHub releases. Verify the checksum.
$ wget https://github.com/chronocoders/ferrous/releases/latest/ferrous-linux-x86_64.tar.gz
$ sha256sum -c ferrous-linux-x86_64.tar.gz.sha256
ferrous-linux-x86_64.tar.gz: OK
$ tar -xzf ferrous-linux-x86_64.tar.gz && chmod +x ferrous
03

Configure

Create a minimal config file. Most defaults are production-ready.
# ferrous.toml
network = "mainnet"
data_dir = "~/.ferrous"
bind = "0.0.0.0:8333"
max_peers = 125
rpc_bind = "127.0.0.1:8332"
# Optional: enable mining
mine = false
coinbase_address = ""
04

Run

Start the node. It will connect to seed nodes and begin syncing.
$ ./ferrous --config ferrous.toml

A blockchain built for the next decade, not the last one.

Every major blockchain in production today relies on ECDSA for transaction signing. ECDSA is broken by Shor's algorithm on a cryptographically relevant quantum computer. The question is not whether this happens, but when.

Ferrous was built from scratch in Rust with post-quantum cryptography as a core requirement, not an afterthought. CRYSTALS-Dilithium replaces ECDSA at the protocol level. There are no legacy keys to migrate, no governance proposals to pass, no community coordination required.

Privacy is the second pillar. On a transparent ledger, every transaction is permanently public. Ring Confidential Transactions make amounts and sender identities confidential by default, without opt-in or separate shielded pools.

Proof-of-Work is the consensus mechanism because it works. Fifteen years of adversarial conditions on the largest decentralized network in existence is a stronger argument than any whitepaper.

Built by a solo developer. Open source. No venture funding. No token sale. No foundation.
